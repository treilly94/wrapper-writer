class wrapper_writer.wrapper_writer.WrapperWriter(method_config_path='./method_config.yml', structure_config_path='./structure_config.yml')¶

The WrapperWriter class contains the details and functionality associated writing a wrapper file based on two configs.

Parameters:
  • method_config_path (str) – The path to the method config file relative to the cwd.
  • structure_config_path (str) – The path to the structure config file relative to the cwd.
structures = {}¶

The dictionary which holds all the information from the structure config.

containers = {}¶

The dictionary which holds all the information from the methods config.

project_root = ''¶

The absolute path to the current working directory.

structure_classes = []¶

The list which holds all the structure classes.

container_classes = []¶

The list which holds all the container classes.

wrappers = []¶

The list which holds all the wrapper classes.

read_configs()¶

This function will read in two yml files and saved them as two dictionaries, containers and structures. It will then get the project root from the structures yml file.

instantiate_structure_class()¶

This function will instantiate the Structure class for each structure within the structures dictionary class parameter. It will store in class within a list.

instantiate_container_class()¶

This function will instantiate the Container class for each container within the container dictionary class parameter. It will store in class within a list.

create_directories()¶

This function will take the structure_classes parameter and called the create_path and create_dir functions for each structure class within the list.

instantiate_wrapper_class()¶

This function will instantiate the Wrapper class for each structure and each container within the structure and container class. It will store these wrapper classes within a list.

run()¶

This function will run the above method in order to produce a wrapper file.

Structure¶

class wrapper_writer.structure.Structure(project_root, path, template, file_name_format)¶

The Structure Class contains the details and functionality associated with a particular structure.

Note

A structure is a group of wrappers that would use the same template, follow the same naming convention, and be placed in the same directory.

Parameters:
  • project_root (str) – This is the absolute path to the project
  • path (str) – The relative path from the project root where the new files will be placed.
  • template (str) – The name of the template file to use.
  • file_name_format (str) – The format of the new file names using PyFormat where a {} represents the file name.

class wrapper_writer.parsers.ScalaParser(config_name='method_config.yml', append_config='w')¶

This ScalaParser class parses a scala file, extracts the method elements and writes them out to a config file :param filename: The name of the file to parse :param config_name: The name of the yaml config file :param append_config: boolean value, True will overwrite an existing file, False will append to file

regex_string = '/\\*\\*([\\s,\\w,\\*,@,\\+\\.,=\'\\[\\]\\-/%]*)\\*/|(\\w+ def|def)\\s(\\w+)\\(([\\w+:\\s,\\[\\]=\\"\\(\\)]*)\\):\\s([\\w\\[\\]]*)'¶

The regex string specific to scala in order to get the docs, params, returns, type and name.

regex_parser(data)¶

This function takes in data as a string, then uses the regex string variable within this class. To find the doc strings, name, parameters, types and return type for each function within the data. It then stores this information as a Method class and saves all the methods as a list. :param data: :return: methods

create_containers(methods, file_path)¶

This function takes the list of Method classes and a list of file paths, if the file path is empty or none, then an error is raised. If there are valid file paths then the container name is found using splits on the file path separator and dots. From there it checks if there are Method Classes within the method list, if there are then a Container class is created and the config is created around that class. Otherwise it prints out which container has no methods

Parameters:
  • methods (List[str]) – This is a list of all the methods class within the file.
  • file_path (str) – This is the file path to the file currently being parsed.
run()¶

This function runs the whole process provided there are files within the file list.
